Automated Slack Notifications & Task Allocation (Make.com)

Make.com (Integromat) ‚Ä¢ Slack ‚Ä¢ Webhooks/Events ‚Ä¢ Workload Balancing

A Make.com scenario that listens for new-page events, posts formatted Slack alerts, and auto-assigns tasks based on current teammate workload. Keeps handoffs fast and distribution fair‚Äîno manual triage.

Overview

Built entirely in Make.com, this automation captures ‚Äúnew page‚Äù events from the source app, enriches metadata (owner, tags, priority), posts a Slack message (Blocks), and assigns the follow-up to the best teammate based on a lightweight load score stored in a sheet/datastore.

Key Features

  • Real-time Slack alerts with page link, owner, and priority badges.
  • Workload-aware auto-assignment using Make routers, filters, and array tools.
  • Manual override via Slack (emoji or button ‚Üí webhook) supported.
  • Idempotency: duplicate-event guards and retry with backoff.
  • Audit notes appended back to the source or a tracker record.

Tech Stack

Make.com (HTTP/Webhooks, Routers, Filters, Variables, Iterator/Aggregator, Data Stores), Slack (Incoming Webhook / Chat:PostMessage), optional Google Sheets or Make Data Store for workload stats, and optional tracker (Notion/Jira/Trello) via Make modules.

How It Works

  1. Trigger: Webhook/Watch module fires on ‚ÄúNew Page.‚Äù
  2. Enrich: Lookup owner/tags; compute priority.
  3. Load score: Pull open items per teammate from Sheets/Data Store; apply cooldown weights.
  4. Route: Router selects the best assignee; filters handle exceptions/overrides.
  5. Notify: Post Slack Block message to the channel with assignments.
  6. Persist: Update tracker + Data Store for audit and future balancing.

Impact

  • Seconds-fast visibility for new work in Slack.
  • Smoother distribution and fewer bottlenecks.
  • Less manual coordination; clearer ownership.
