Mail Merge (Gmail + Google Sheets)

Python/Google Apps Script • Google APIs • Gmail Drafts • Personalization

A custom Mail Merge app that reads a Google Sheet (websites + emails), personalizes content, and automatically creates Gmail drafts at scale. Designed to send ~50 tailored emails in just a couple of minutes.

Overview

I built a lightweight Mail Merge tool that connects Google Sheets and Gmail via Google APIs. From a sheet of target websites and emails, it generates personalized email bodies (using fields like name, company, and website) and saves them as Gmail drafts for quick review and sending.

Key Features

  • Reads recipients and context from a Google Sheet.
  • Builds personalized subject lines and email bodies with placeholders (e.g., {{name}}, {{company}}, {{website}}).
  • Creates Gmail drafts automatically—human review before send.
  • Idempotent runs (skips or marks already drafted rows to avoid duplicates).
  • Basic rate limiting to stay within Gmail/Google API quotas.
  • Runtime: roughly 50 emails in a few minutes end-to-end.

Tech Stack

Google APIs (Sheets API, Gmail API) orchestrated with Python or Google Apps Script (depending on environment). Data source is a structured Google Sheet containing personalization fields.

How It Works

  1. Prepare a Google Sheet with columns like Name, Email, Company, Website, and Status.
  2. Authenticate with Google (OAuth / Apps Script scopes).
  3. Read each row and render the email using a simple template.
  4. Call the Gmail API to create a draft (subject + body) for that recipient.
  5. Update the Sheet row Status (e.g., Drafted/Sent) to prevent duplicates.
  6. Manually review drafts in Gmail and send in batches as needed.

Impact

  • Cut outreach time dramatically—about 50 emails in minutes.
  • Higher reply rates thanks to relevant, personalized lines.
  • Lower risk: drafts allow final human edits before send.
