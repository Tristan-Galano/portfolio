Basic POS with Debt Tracking

Python ‚Ä¢ CLI/Script ‚Ä¢ Debt Ledger ‚Ä¢ Sales Summary

A lightweight Point of Sale tool that records sales, tracks customer debts, and generates simple sales summaries. Built to keep daily transactions clear and outstanding balances visible.

Overview

The project provides core POS actions (add items, record sales, manage customers) and introduces a debt tracking layer so you can log partial payments and keep an up-to-date balance per customer.

Key Features

  • Record sales with date/time and totals.
  • Create customers and track their running debt/balance.
  • Apply payments (partial or full) and auto-recalculate remaining balance.
  • Simple sales report to view daily/overall totals.
  • Lightweight data model that‚Äôs easy to extend (e.g., inventory later).

Tech Stack

Python for the core logic. Data is stored in simple structures/files (CSV/JSON/SQLite ‚Äî depending on your latest commit; feel free to update this line).

How It Works

  1. Add or select a customer.
  2. Enter items/amount and save the sale.
  3. If unpaid, the difference is recorded as debt.
  4. When the customer pays later, record the payment and the balance updates automatically.

Impact

  • Clear visibility on outstanding balances per customer.
  • Fewer manual errors when tracking partial payments.
  • Fast end-of-day summaries for sales and collections.
